**EditorId type and EDITOR_CONFIGS constant:**
```typescript
export type EditorId = "cursor" | "claude-code" | "windsurf" | "vscode" | "opencode";
```

Define `EDITOR_CONFIGS` as a record mapping each EditorId to:
- `name`: display name (e.g., "Cursor", "Claude Code", "Windsurf", "VS Code", "OpenCode")
- `configType`: "file" or "cli"
- `configPath`: function `(projectDir: string) => string` returning the config file path. Use:
  - Cursor: `path.join(projectDir, ".cursor", "mcp.json")` (project-level, per D-context: `.cursor/mcp.json`)
  - Claude Code: no configPath (cli-based)
  - Windsurf: `path.join(os.homedir(), ".codeium", "windsurf", "mcp_config.json")` (global)
  - VS Code: `path.join(projectDir, ".vscode", "mcp.json")` (project-level)
  - OpenCode: `path.join(projectDir, ".opencode", "mcp.json")` (project-level)
- `detectPaths`: array of paths to check for existence to detect the editor is installed. Use:
  - Cursor: check `path.join(projectDir, ".cursor")` dir exists OR `path.join(os.homedir(), ".cursor")` exists
  - Claude Code: check if `claude` binary exists on PATH via `execSync("which claude")` wrapped in try/catch
  - Windsurf: check `path.join(os.homedir(), ".codeium", "windsurf")` dir exists
  - VS Code: check `path.join(projectDir, ".vscode")` dir exists
  - OpenCode: check `path.join(projectDir, ".opencode")` dir exists

**`detectEditors(projectDir: string): EditorId[]`** — iterate EDITOR_CONFIGS, check detectPaths with `fs.existsSync`, return array of detected editor IDs.

**`buildMcpServerEntry(readonlyKey: string)`** — returns the MCP server config object for context-company:
```typescript
{
  "type": "http",
  "url": "https://api.thecontext.company/mcp",
  "headers": {
    "Authorization": `Bearer ${readonlyKey}`
  }
}
```

**`writeMcpConfig(editorId: EditorId, projectDir: string, readonlyKey: string): void`** — for file-based editors:
1. Get the config path from EDITOR_CONFIGS
2. If config file exists, read and parse it as JSON
3. Ensure `mcpServers` key exists (create if not)
4. Set `mcpServers["context-company"]` to `buildMcpServerEntry(readonlyKey)` — this adds/updates only the context-company entry, preserving all other servers (MCP-05)
5. Write back with `JSON.stringify(parsed, null, 2)` + newline
6. Create parent directories with `fs.mkdirSync(dir, { recursive: true })` if needed

**`runClaudeMcpAdd(readonlyKey: string): { success: boolean; error?: string }`** — for Claude Code:
1. Build command: `claude mcp add --transport http context-company https://api.thecontext.company/mcp --header "Authorization: Bearer ${readonlyKey}"`
2. Run via `execSync` with stdio piped, timeout 10_000
3. Return `{ success: true }` on success, `{ success: false, error: message }` on failure

Use `import { execSync } from "node:child_process"`, `import fs from "node:fs"`, `import path from "node:path"`, `import os from "node:os"`.

Follow project conventions: double quotes, 2-space indent, [TCC] prefix for console.error, JSDoc on all exports.

Export `setupMcpStep` implementing the `Step` interface:

`shouldRun(ctx)`:
- Return `false` if `ctx.keyProvided` is true (--key mode skips MCP, per AUTH-06/CONTEXT decisions)
- Return `false` if `!ctx.readonlyKey` (no readonly key available)
- Return `false` if `ctx.completedSteps.includes("setup-mcp")` (idempotency)
- Otherwise return `true`

`run(ctx)`:
1. Show benefits explanation (MCP-07) using `p.log.info()`:
   ```
   p.log.info(
     pc.cyan("MCP connects your AI coding tools to production observability.\n") +
     "Your editor can now query prod runs, find failures, and search insights."
   );
   ```

2. Call `detectEditors(ctx.installDir)` to get detected editors

3. Build options array from all EDITOR_CONFIGS entries. Each option: `{ value: editorId, label: editorName, hint: detected ? "detected" : undefined }`

4. Use `p.multiselect()` to show editor selection:
   ```typescript
   const selected = await p.multiselect({
     message: "Which editors should we configure for MCP?",
     options,
     initialValues: detectedEditors,  // pre-check detected ones (MCP-02)
     required: false,
   });
   ```

5. If user cancels (p.isCancel(selected)) or selects none, return `{ status: "skipped", message: "MCP setup skipped" }`

6. For each selected editor:
   - If `configType === "file"`: call `writeMcpConfig(editorId, ctx.installDir, ctx.readonlyKey!)` wrapped in try/catch. Log success with `p.log.success(`Configured ${name}`)` or warn on error.
   - If `configType === "cli"` (claude-code): call `runClaudeMcpAdd(ctx.readonlyKey!)`. Log success or warn on error.

7. Push `"setup-mcp"` to `ctx.completedSteps`

8. Return `{ status: "completed" }`
